# CMYK

# PALETA FÍSICA (CMYK MODE)




  **El Mundo Físico No Emite Luz.** Cuidado profundo: Un novato se equivoca gravemente asumiendo una igual conversión; en tu brillante monitor quemas luz <code>RGB</code>, 
  mientras que en lonas, papelería y vinil automotriz el plotter escupe forzosamente densas tintas sustractivas <code>CMYK</code> que absorben agresivamente la luz ambiental.


---

:::warning 🚨 COMPATIBILITY BUG: OUT_OF_GAMUT ERROR
Las luminosas y letales variantes corporativas **(Rosa Ajolote y Cyan Eléctrico)** son radiantes frecuencias puramente electromagnéticas sintéticas de computadora. Jamás de los jamases un plotter regular 
y anticuado podrá igualar su impacto fosforescente mezclando solo los clásicos rodillos aburridos CMYK comerciales de escuela. Tu PDF _RGB_ masivo se achatará inevitablemente de inmediato, vomitando sucios plastones visuales sin brillo ni energía. 
:::



Para salvar la vida a tus asombrosos diseños impresos (`Offline Deploy`), hemos reingenierizado el código genético nativo en **La Paleta CMYK (El "Hard Patch")**. Estas combinaciones CMYK rigurosas en los porcentajes jamás tienen el descabellado objetivo de igualar tu brillante Macbook, pero nos garantizan exprimir y _Hackear_ matemáticamente la formulación porcentual de cuatricromía cruda al límite de la química de los plotters 
hasta escupir la gota de tinta más pesada y rabiosa posible reteniendo vigor.

### ⚖️ Jerarquía de Contraste (Estándar WCAG)

Para no sangrar la retina ocular de nuestros usuarios al leer bloques documentales:

1. **Lectura Óptima Total:** **Blanco Consola** directo sobre la masiva **Obsidiana Digital**. Es la relación estricta rey superior a `15:1` de contraste.
2. **Sistema de Destacados (Highlighting):** Variables ácidas como **Rosa Ajolote** y **Cyan Eléctrico** bailan sobre lienzos oscuros SÓLO para enfatizar microdatos: *Labels, Strings, H2 o Iconos mágicos*. Jamás para un párrafo largo humano.
3. **Estado de Botones (CTAs):** Regla de la Sangre Negra: Si fundes un botón vivo (`Verde, Rosa, Cyan`), la letra interna debe ser cruelmente **Obsidiana**. Nunca la pintes de blanco o explotará visualmente.

### 🛑 Error Fetal: Combinaciones Prohibidas (`Syntax Errors`)

Para sortear las horribles "Vibraciones Cromáticas" y el look pasivo novato, queda penalizado por sistema la inyección del siguiente código visual:

1. **Rosa vibrante sobre Verde radiante:** Destrucción óptica total. Sus frecuencias casi chocan matando el enfoque humano (Efecto de Moiré Psicológico).
2. **Cyan sobre Blanco:** Contraste matemáticamente muerto (`< 3:1`). El texto sufre lavado y simplemente desaparece en un mar de luz blanca inútil.
3. **Texto Blanco sobre Background Verde Terminal:** Un crimen de accesibilidad. El verde brillante se come los pixeles blancos. **Tira Letras oscuras encima del Verde.**

## 💻 Aplicación de Inyección en Código Base (`UI`)

### Las Variables de Ambiente CSS (CSS Vars)
Toda la arquitectura del sistema front-end de `CODE ME` debe estar referenciada obligatoriamente a CSS Tokens. 
Al inyectar paletas semánticamente logras velocidad inaligerable.

```css
/* ROOT: Declarando Variables de Entorno del Sistema */
:root {
  /* Lienzo */
  --cm-obsidiana: #0F172A; 
  --cm-blanco-consola: #F8FAFC; 
  
  /* El Fuel (Acentos) */
  --cm-rosa-ajolote: #FF2A6D; 
  --cm-cyan-electrico: #00F0FF; 
  --cm-verde-terminal: #05FF00; 
  
  /* Estados UI Ocultos */
  --cm-border: rgba(248, 250, 252, 0.15); /* Bordes UI transparentes */
  --cm-selection: rgba(0, 240, 255, 0.3); /* Swipe mágico */
}

/* COMPONENT: Botón Primario de Fuego */
.btn-core {
  background-color: var(--cm-rosa-ajolote);
  /* Regla de Contraste #3 aplicándose: Texto oscuro encima */
  color: var(--cm-obsidiana); 
  text-transform: uppercase;
  font-family: 'JetBrains Mono', monospace; /* Tipo Comando */
  border: 1px solid transparent;
  transition: all 0.2s ease;
}

/* COMPONENT: Hightlight de Código Activo */
.code-highlight_success {
  color: var(--cm-verde-terminal);
  border-left: 2px solid var(--cm-verde-terminal);
  padding-left: 1rem;
}

/* COMPONENT: Hackeando el ratón del usuario */
::selection {
  background: var(--cm-selection);
  color: var(--cm-blanco-consola);
}
```

# ERGONOMIA

# ERGONOMÍA VISUAL Y SALUD UX




  **Development-First Approach.** La identidad entera de <strong>CODE ME</strong> no fue diseñada caprichosamente porque se viera "bonita". 
  Reconocemos que nuestros usuarios críticos (programadores, estudiantes de código, y analistas) 
  pasan brutales periodos prolongados quemándose las retinas frente a monitores emisivos.


---

### 👁️ Reducción de Fatiga Visual (Astenopía Tech)
El uso dictatorial de nuestra gruesa capa de **Obsidiana Digital (`#0F172A`)** como fondo permanente, aplasta radical y eficazmente la dañina emisión descontrolada de luz azul (Blue-light emission) y frena el brillo excesivo de la pantalla en los escritorios.

- **El Fix Fisiológico:** Esto genera directamente un enorme descanso biológico y menor esfuerzo constante del músculo ciliar ocular de los alumnos. Previniendo dramáticamente el ardor ocular, la vista borrosa progresiva y las clásicas migrañas y dolores de cabeza frontales tras épicas de jornadas de 8 horas tirando código.

### ⭕ Prevención del Efecto "Halo" (Astigmatismo)
Para millones de usuarios con astigmatismo leve o de alto grado, un texto blanco absoluto 100% puro radiando fotones ciegamente frente a un negro vacío absorbente, no se lee nítido, sino que les genera físicamente un doloroso efecto visual de "Difuminado" o halo parpadeante.

- **El Parche UX:** Por esto mismo creamos matemáticamente el **Blanco Consola (`#F8FAFC`)**. Al rebajar fríamente un par de puntos la pureza RGB, logramos entintarlo ligeramente, suavizando inmensamente el contraste cortante extremo. Permitiendo una lectura de párrafos mil veces más relajada y nítidamente filosa sobre el canvas puro.

### 🚥 Control de la Vibración Cromática Óptica
Ciertas combinaciones de los colores luz de alta frecuencia (Como nuestro ardiente Rosa Ajolote directamente puesto y haciendo frontera sobre el Verde Terminal) provocan una furiosa sobre-estimulación química de los fotorreceptores inmediatos de la retina humana.

- **Warning System:** Esto causa a corto plazo una irreal sensación fisiológica de "movimiento ciego" o desagradable resplandor vibratorio (Moiré de bordes). Por eso mismo, queda prohibidísimo por manual empatar áreas inmensas de choque de colores primarios neón vivos sin aplicar _padding_ o filetes ciegos negros de Obsidiana para apaciguar el choque.

### ☀️ El "Modo Blueprint" (Protocolo de Día)
Aunque somos una pesada marca de culto al `Dark Mode` (código de noche), en entornos físicos con altísima radiación lumínica natural directa en campo (Impresión Offset o Exteriores OOH), obligar al ojo humano ciego a escudriñar formas negras absorbiendo luz causa un doloroso esfuerzo ocular inverso.

- **Switch Autorizado:** Se permite tácticamente el _override_ al uso pleno táctico de la variante fotográfica clara (Nuestro **Modo Blueprint Papel de Ingeniería Blanco**) para publicidades que se exponen en calle en pleno sol radiante cenital, protegiendo totalmente al usuario del efecto de deslumbramiento urbano ciego.

---

### 🧮 Diferentes Capacidades Visuales (Color Blindness)
Es de suprema importancia en una empresa de desarrollo web inclusiva puramente digital auditar y entender rigurosamente cómo es percibida por los padres de familia nuestra paleta si sufren de alteraciones crónicas genéticas en la visión del color cromático regular.

Nuestra meta maestra técnica es que nuestra jerarquía publicitaria no se desplome, sino que su robustez se base infaliblemente en la profunda diferencia de **Luminosidad Perceptual Cruda** (Blancos VS Negros) y no solamente en la dependencia ingenua de puros Tintes Bonitos en el color.

# NOTAS

# NOTAS IMPORTANTES (TIPS)



---

### ¿Por qué diablos mis colores tienen dos códigos HEX distintos?

Es tu culpa en absoluto. Es una fantástica pregunta que se hacen todos los alumnos de primeros ciclos. La respuesta lógica técnica está en la infinita brecha universal que existe entre emitir **Luz (`RGB`)** y plasmar **Tintas Químicas (`CMYK`)**.

#### 1. El Runtime Mode Físico (Tintas CMYK)
- **Concepto:** Todos los clásicos colores que pides que se impriman en un enorme vinil automotriz o en el libro oficial de la escuela (como nuestra versión `#E6007E`) son inherentemente **reflectores muertos de luz**.
- **La Física:** Son tintas orgánicas sucias y apagadas forzosamente por pura química natural, porque la tinta cruda _absorbe luz muerta_ en lugar de re-emitirla al ojo humano.
- **Funcionamiento Matemático:** Trabajan burdamente restándole puros niveles de luminosidad al pobre papel encapado blanco maestro. (Cian + Magenta + Amarillo + el temido Key Plate Negro).

#### 2. El Runtime Mode Pantalla (Pantallas RGB)
- **Concepto:** Por el contrario total; los impresionantes y ciegos colores fosforescentes que amas ver en tu monitor retina display nuevo (como nuestro `#FF2A6D`) son descarados **emisores violentos de luz fotónica**.
- **La Física:** Son vibrantes al máximo, quemantes, brillantes y literalmente "Electric". Exudan radiación. No hay tinta, son diodos de led quemados.
- **Funcionamiento Matemático:** Funcionan lógicamente _sumando frecuencias puras_ de fotones calientes (Vectores Rojo + Verde + Azul hasta llegar al blanco máximo de la ceguera).


### 💡 TL;DR (Resumen de Producción)
Nosotros siempre llamaremos en nuestro front-end al majestuoso **código primario de Luz (`RGB`)** para levantar la fantástica Matrix Web de `Code Me` y las pantallas táctiles. 

Nos ahorcaremos y tendremos que usar dócilmente el terrible **código secundario opaco (`CMYK`)** forzoso para intentar imprimir lonas espectaculares enormes en avenida, o simples playeras promocionales de algodón barato publicitario. 

Si cometieras la locura de intentar mandar imprimir por necedad tu `#FF2A6D RGB puro` directamente y sin modificar variables en una impresora de inyección EPSON; la estúpida máquina al verse imposibilitada de lanzar diodos de luz, simplemente la ahogará en su limitado perfil de tinta, destruyendo nuestro Neón y entregándote deprimentemente el peor color rosado pastel ocre sucio y sin alma jamás antes visto en la historia universal. Por eso mismo exacto programamos el _"Parche Modesto de Valores CMYK"_, simulando manualmente la química estricta de compensación para emparejar colores oscuros.

:::warning 🚨 FATAL ERROR: REGLA DE DOS AMBIENTES
Si el gráfico vivirá nativamente atrapado en tu celular o pantalla OLED; usas **Luz (Paleta RGB 1)**. <br/>
Si obligatoriamente existirá su soporte en el mundo de papel celulosa física; usas **Tinta (Paleta CMYK 2)**. <br/>
Jamás crees archivos cruzándolas mutuamente, la consola lo registrará como error.
:::

El programador (nuestra `JetBrains Mono` nativa de IDE) o similares monolíticas como Fira Code. Esta familia domina brutalmente Menús de Navegación, Footers Crudos y Estadísticas numéricas.
- **Fallo Tolerado:** Si por razones de compatibilidad del sistema receptor no puede correr la familia pesada o la JetBrains, por regla se fuerza un _fallback_ a cualquier Monoespaciada nativa dura del Sistema Operativo (`Courier New HD`, `Consolas`). Pero nunca a fuentes redondeadas (Tipo _Comic_ o _Papyrus_) a menos que sea un meme a propósito.

#### B. Legibilidad en Pantallas (El Balance `p`)
- **Regla Estricta:** Para los pesados bloques documentales masivos (Decenas de líneas consecutivas para padres de familia), JAMÁS uses una fuente monoespaciada para el _Body Text_. Agotarás la retina y paciencia del padre usuario. Obligado está usar tu cálida humana **Montserrat** (o equivalente geométrica libre).
- **Control Fino:** El crucial interlineado (`line-height`) flotando para bloques densos nunca puede declararse inferior mecánicamente a **`1.6em`** para Dark Mode. Esto obedece ciegamente a estrictas reglas de "Leyes UX de Salud Visual en modo noche", evitándonos ese horrendo _apelmazamiento cegador_ de píxeles pesados blancos ardiendo sobre pixeles negros masivos.

#### C. Uppercase & Tracking (Botones de Comandos)
- **Regla:** Queda prohibido el tratamiento pasivo "Click aquí" en los Botones `[ CTAS / Enlaces directos ]`. Los Botones deben simular en mayúscula sostenida **(UPPERCASE)** ser rudos comandos del DOS para el cursor, abriendo ligeramente en `CSS` el aire dentro de sus letras (`letter-spacing: +0.05em a +0.1em`). 

---

### 🛑 Anti-Patrones de Compilación (Warning_Flags)

1. **NO MEZCLAR `> 2` TIPOGRAFÍAS (`var overload = false`)** 
   - Está penado por diseño cargar más de DOS familias diferentes de letras sobre el mismo frame/cuadro de vista. Saira masiva y Montserrat limpia para humanos. O JetBrains robótica y Saira si requieres un marco super _Hacker_ técnico. Jamás las 3 al mismo tiempo compitiendo en jerarquía; eso es ruido de aficionado y debilita dolorosamente tu layout gráfico en milisegundos.

2. **CERO USOS DE FUENTES SERIFS (`!important`)**
   - Literalmente proscrito a fuego el uso visual de tipografías antiguas con _patines o adornos_ clásicos (Las _Times New Roman_, las _Garamond_) incluso si un texto nos lo roga en "citas formales escolares". Rompen estrepitosamente la atmósfera de inmersión Cyberpunk Tecnológica brutal que hemos creado cuadro a cuadro. Aquí nada es época de la imprenta vieja; todo es vectores digitales y Neón.

3. **CERO ITÁLICAS INNECESARIAS (Falsos énfasis)**
   - La cursiva clásica (`<i>` / `font-style: italic`) ha quedado extinta conceptualmente como medio real para dar protagonismo de lectura general. Aquí los programas no usan cursiva de máquina de escribir inclinada; aquí usamos destellos puros. Si en tu párrafo necesitas realzar fuertemente una idea, no la tuerzas: **la inyectas furiosamente de Semi-Bold (#FF2A6D - Rosa Matrix)** o encierras el texto lúdicamente entre puras marcas de comentarios del IDE estilo `/* Highlight System */` para forzar la sonrisa del que sabe de lenguaje técnico.

# PRINCIPAL

# FUENTE PRINCIPAL (`H1`)




  **La Voz de Mando Sistemático**. La tipografía que recibe el primer impacto visual en la arquitectura de <strong>CODE ME</strong>. 
  Pura presencia brutalista e industrial que construye bloques inamovibles.


---

### 🏗️ Familia Elegida: Saira (Extra Expanded)
[Saira](https://fonts.google.com/specimen/Saira) es una portentosa tipografía sans-serif de Google Fonts que se caracteriza por su brutal capacidad y solvencia para impactar visualmente el espacio con pura gravedad. 
Al usar **tácticamente** su versión _“Extra Expanded”_ (Estrecho extra ancho), logramos emular instantáneamente el _Vibe Retro-tech_ de aquellas pesadas matrices de pantallas antiguas. 
Es una fuente 100% estructural; luce idéntica a los bloques físicos de un rascacielos digital pesado.


    <FontShowcase 
        fontFamily="Saira, sans-serif" 
        defaultText={"if (idea) { bitxo.build(); }"} 
        allowedWeights={[800, 900]} 
    />


### 🛠️ Configuración (Specs)
- **Pesos (`font-weight`):** Exclusivamente los más aplastantes. `Black (900)` o `ExtraBold (800)`.
- **Ancho (`font-stretch`):** Literalmente imperativo configurarlo a `extra-expanded` (o elegir la familia Saira Extra Condensed en su defecto si tu software no procesa font-stretch variable).
- **El Rol:** Actúa psicológicamente como _El Muro de Contención_. 
- **Tracking:** Generalmente un poco más apretado (Tight `letter-spacing: -2%`) para amarrar las letras unas dándose soporte contra otras como un puro bloque en vez de palabras separadas flotando solas.

### 📍 Puntos de Montaje Aprobados (Deploy)
- Uso dictatorial y sin excepciones para **Títulos y Encabezados Mayores (`H1`)**.
- **Regla Estricta del Backend:** El `H1` Saira SIEMPRE se imprime en letras `MAYÚSCULAS` Sostenidas. Nunca capitalizadas ("Abc") o minúsculas. 
- La fuente principal para cualquier texto pesado sobre playeras o Merchandising.
- En macro-pantallas corporativas o diapositivas donde necesitemos que una sola frase acapare todo el canvas.

### 💻 Implementación en Código

```css
/* Injecting Root Typographic Scale */

h1, .saira-heavy-title {
    font-family: 'Saira', sans-serif; 
    font-weight: 900;
    font-stretch: extra-expanded; 
    text-transform: uppercase;
    letter-spacing: -0.02em; /* Pegando los caracteres de sistema */
    line-height: 1.1; /* Rompiendo el interlineado flojo */
}
```

# SECUNDARIA

# FUENTE SECUNDARIA (`<P>`)




  **La Voz Humana**. La tipografía nativa con curvas geométricas diseñada para que los humanos reales (padres de familia corporativos y alumnos leyendo código) **nos entiendan ágilmente** sin forzar la vista en pantallas saturadas.


---

### 💬 Familia Elegida: Montserrat
[Montserrat](https://fonts.google.com/specimen/Montserrat) es nuestra navaja suiza de la legibilidad. Es una fuente sin patines (_sans-serif_) tremendamente moderna, pulcramente geométrica pero amable a la vista. 
Su rol supremo en el canvas gráfico es **traducir y rebajar** la arrolladora intensidad visual bruta de nuestras otras dos tipografías (La brutalista _Saira_ y la robótica _JetBrains_), permitiendo a los usuarios consumir párrafos largos, explicaciones complejas y legales corporativos sin agotarse en fracciones de segundo.


    <FontShowcase 
        fontFamily="'Montserrat', sans-serif" 
        defaultText={"if (idea) { return bitxo.build(); }"} 
        allowedWeights={[400, 600, 700]} 
    />


### 🛠️ Configuración (Specs)
- **Pesos Aprobados (`font-weight`):** `SemiBold (600)` para hacer resaltar los links, negrillas de énfasis (`<b>`) y subtítulos ligeros. O el balanceado `Regular (400)` estricto para cuerpos de texto.
- **El Rol:** El balance. La claridad mental (UX) y la lectura sostenida.
- **Tracking:** Relajado. Nunca debe ir apretada. Deja que las letras respiren a `0em` o `+0.01em` para maximizar la lectura horizontal en resoluciones móviles.

### 📍 Puntos de Montaje Aprobados (Deploy)
¿Dónde implementar este recurso textual?
- Cuerpos de texto masivos (Body text, Documentación general y `Text-areas`).
- Textos y _Footers_ Legales.
- Documentos corporativos y PDF de alianzas estratégicas dirigidos a socios (Business to Business).

### 💻 Implementación en Código

```css
/* Injecting Soft Typographic Scale */

p, span, .body-text {
    font-family: 'Montserrat', sans-serif; 
    font-weight: 400; /* Regular base */
    line-height: 1.6; /* Aire vital entre lineas */
    color: var(--cm-blanco-consola); /* Contraste Dark Mode */
}

b, strong {
    font-weight: 600; /* Pop-out SemiBold */
}
```
